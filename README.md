# README.md

Данный программный продукт является объектом выпускной квалификационной работы (ВКР) по специальности 09.02.07 «Информационные системы и программирование». Система спроектирована для автоматизации процессов ценообразования и управления товарным ассортиментом магазина электронной техники, объединяя функции интернет-магазина и интеллектуального модуля динамического перерасчёта цен.

## 1. Технологический стек (Stack)

Система реализована на базе актуальных технологий веб-разработки и управления данными:
- Платформа: Кроссплатформенное веб-приложение (Windows 10/11, Android 8.0+, macOS, Linux).
- Клиентская часть: TypeScript, Vue 3.5 (Composition API), Vite, SCSS, Tailwind CSS, Lodash, VueUse.
- Серверная часть: Python 3.10+, FastAPI, SQLAlchemy (ORM), Pydantic, PostgreSQL, Docker.
- База данных: PostgreSQL 15 (реляционная модель, 16 таблиц).
- Инструменты: Visual Studio Code, Git, Figma, Draw.io, Docker Compose.
- Тестирование: vitest (unit), Cypress (e2e), Postman (API).

## 2. Архитектурные решения

Проект придерживается клиент-серверной архитектуры с разделением ответственности (Separation of Concerns) и использованием REST API:
- **Клиентская часть (SPA):** построена на компонентном подходе Vue 3.5 с использованием Composition API. Реализованы реактивные хранилища состояния (Pinia-подобный подход), маршрутизация (Vue Router), оптимистичное обновление интерфейса и дебаунсинг запросов.
- **Серверная часть (FastAPI):** реализует слой API с автоматической валидацией через Pydantic, асинхронную обработку запросов, middleware для аутентификации (JWT) и авторизации (ролевая модель). Используется паттерн Repository для доступа к данным.
- **База данных:** физическая модель IDEF1X включает сущности: `user`, `roles`, `products`, `categories`, `brands`, `analogs`, `cart_items`, `orders`, `order_items`, `recalculate_history`, `recalculations`, `log_settings` и журнальные таблицы. Обеспечена ссылочная целостность, каскадное удаление, индексы для оптимизации запросов.
- **Модуль ценообразования:** выделен в отдельный сервис (`recalculations.py`), поддерживает несколько математических стратегий пересчёта (абсолютное/относительное изменение, установка фиксированной цены, пересчёт по средней цене аналогов). Каждая операция логируется в таблицу `recalculate_history` с параметрами, инициатором и временем выполнения.
- **Журналирование изменений:** реализовано триггерное логирование в таблицы `*_log` (user_log, product_log и др.) с настраиваемым временем и количеством хранения записей.

## 3. Ключевые функциональные модули

### 3.1. Модуль аутентификации и управления пользователями
Реализована система регистрации и авторизации с хешированием паролей (bcrypt). Поддерживается ролевая модель:
- **Гость (guest):** только просмотр каталога.
- **Пользователь (user):** добавление в корзину, оформление заказов, просмотр профиля.
- **Модератор (moderator):** CRUD товаров, категорий, брендов; управление статусами заказов.
- **Суперпользователь (superuser):** все права, включая управление ролями, просмотр истории перерасчётов, выполнение пересчёта цен, настройку автоматических пересчётов.

### 3.2. Модуль интернет-магазина (каталог, корзина, заказы)
- **Каталог товаров:** отображение в виде таблицы или карточек. Реализованы фильтрация по категориям/брендам/диапазону цен, полнотекстовый поиск по названию и описанию, сортировка по цене, названию, рейтингу.
- **Корзина:** добавление/удаление товаров, изменение количества, полная очистка. Данные корзины сохраняются между сессиями (localStorage + синхронизация с БД).
- **Оформление заказа:** создание заказа с генерацией уникального кода получения, очистка корзины, изменение статуса заказа (pending, ready, completed, cancelled).

### 3.3. Модуль автоматического перерасчёта цен
Поддерживаются следующие математические стратегии (реализованы в `recalculations.py`):
- **Относительное изменение:** увеличение или уменьшение цены на фиксированную сумму (рубли) или процент.
- **Фиксированная цена:** установка для всех выбранных товаров единой цены.
- **Пересчёт по аналогам:** на основе средней цены аналогов из таблицы `analogs` (с возможностью добавить наценку или скидку в процентах). Дополнительно может использоваться минимальная цена аналога со скидкой.

Пользователь (администратор) выбирает товары через интерфейс (фильтры, множественный выбор), задаёт параметры пересчёта, система выполняет операцию и сохраняет запись в `recalculate_history`. Также предусмотрена возможность создания автоматических пересчётов по расписанию (таблица `recalculations`).

### 3.4. Модуль экспорта данных и отчётности
Для таблиц с данными (товары, пользователи, заказы, бренды, категории, аналоги, история перерасчётов) реализован экспорт в формате XLSX (библиотека `xlsx` на клиенте). Пользователь с соответствующими правами может выгрузить текущее представление таблицы (с учётом фильтров и сортировки) в файл для дальнейшего анализа.

### 3.5. Модуль аудита и настройки логирования
- Автоматическое журналирование операций INSERT, UPDATE, DELETE в таблицах `user`, `category`, `brand`, `order`, `product` (хранятся в соответствующих `*_log` таблицах с типом операции и временной меткой).
- Настройки удержания логов (`log_settings`): для каждой таблицы задаётся время хранения в минутах и лимит количества записей. Фоновый процесс (или административная команда) очищает устаревшие логи.
- Просмотр логов через интерфейс администратора (фильтрация по операции, дате, поиск).

## 4. Информационная безопасность и защита данных

- **Аутентификация и авторизация:** JWT-токены с коротким временем жизни, хеширование паролей (bcrypt). Ролевая модель проверяется на каждом защищённом эндпоинте через dependency в FastAPI.
- **Валидация данных:** входные данные проверяются Pydantic-схемами на сервере; на клиенте используется дополнительная валидация форм. Предотвращение SQL-инъекций через ORM (SQLAlchemy).
- **Защита от потери данных:** транзакционная обработка операций с БД (атомарность). Автосохранение корзины в localStorage и синхронизация с сервером при авторизации.
- **Логирование действий:** все критичные операции (изменение цен, удаление товаров, изменение ролей) сохраняются в журналах с указанием пользователя и времени.
- **Обработка ошибок:** на клиенте отображаются понятные сообщения об ошибках (недостаточно прав, неверный формат, дублирование). Сервер возвращает стандартизированные HTTP-статусы с описанием.

## 5. Инструкция по развертыванию

### Требования к окружению
- Установленный Docker Engine и Docker Compose.
- Node.js 18+ (только для клиентской разработки, в production клиент собирается статически).
- Браузер (Chrome, Яндекс.Браузер, Firefox) для доступа к приложению.

### Шаги для локального запуска (режим разработки)

1. Распакуйте архив проекта в любую директорию.
2. **Запуск серверной части и базы данных:**
   \`\`\`bash
   cd app/backend
   docker-compose up -d
   \`\`\`
   После запуска сервер будет доступен по адресу `http://localhost:8000`. Интерактивная документация API (Swagger) — `http://localhost:8000/docs`.
3. **Запуск клиентской части:**
   \`\`\`bash
   cd app/frontend
   npm install
   npm run dev
   \`\`\`
   Приложение будет доступно по адресу `http://localhost:5173`.
4. **Тестовые учётные данные** (предзагружены в БД):
   - Пользователь: `user@example.com` / `user1234` (роль "пользователь").
   - Модератор: `moderator@example.com` / `mod1234` (роль "модератор").
   - Администратор: `admin@example.com` / `admin1234` (роль "суперпользователь").

> **Примечание:** Для production-сборки клиент компилируется в статические файлы и раздаётся через Nginx (конфигурация в `docker-compose.prod.yml`). Миграции базы данных выполняются автоматически через Alembic при первом запуске.

## 6. Управление (Интерфейс)

| Элемент управления | Действие |
|-------------------|----------|
| Левая кнопка мыши (ЛКМ) | Навигация по меню, нажатие кнопок, выбор товаров, редактирование ячеек таблиц (двойной клик или кнопка «Редактировать») |
| Кнопка «Добавить» (в таблицах) | Открытие боковой панели для создания новой записи |
| Кнопка «Экспортировать» | Скачивание текущей таблицы в формате XLSX |
| Поле поиска / фильтры | Ввод текста, выбор категорий/брендов, задание диапазона цен; фильтрация происходит с debounce 300 мс |
| Чекбокс множественного выбора | Выбор нескольких записей для массового удаления или пересчёта цен |
| Кнопка «Пересчитать цены» (админ) | Открытие диалога с настройками пересчёта (тип изменения, значение) |
| Кнопка «Выйти» (в профиле) | Завершение сессии, перенаправление на страницу входа |

Все табличные интерфейсы поддерживают сортировку по любому столбцу (клик по заголовку) и пагинацию (по 20 записей на страницу). Реализовано оптимистичное обновление: после редактирования/удаления записи позиция прокрутки не сбрасывается.
